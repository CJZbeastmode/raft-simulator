## Session 4: `Start()` — Leader Accepts a Command and Sends It

**Goal:** Implement `Start()` so the leader appends a command to its own log, then replicates it immediately (not waiting for the next heartbeat tick).

---

## Step 1 — Implement `Start()` (15 min)

Find your stub:

```go
func (rf *Raft) Start(command interface{}) (int, int, bool) {
    index := -1
    term := -1
    isLeader := true
    // Your code here (3B).
    return index, term, isLeader
}
```

Replace with:

```go
func (rf *Raft) Start(command interface{}) (int, int, bool) {
    rf.mu.Lock()
    defer rf.mu.Unlock()

    // Only the leader accepts new commands
    if rf.state != "leader" {
        return -1, rf.currentTerm, false
    }

    // Append to own log immediately
    entry := LogEntry{
        Term:    rf.currentTerm,
        Command: command,
    }
    rf.log = append(rf.log, entry)
    index := len(rf.log) - 1
    term := rf.currentTerm

    // Trigger replication immediately, don't wait for next heartbeat tick
    go rf.sendHeartbeats()

    return index, term, true
}
```

Three things happening here:
- Leader appends to its own log first (it's always the first to have it)
- Returns the index where the command will appear if committed
- Kicks off replication immediately — otherwise the tester times out waiting

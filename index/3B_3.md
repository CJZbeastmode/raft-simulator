## Session 5: Advance `commitIndex`, Apply Loop → `applyCh`

**Goal:** Leader detects when a majority has replicated an entry and advances `commitIndex`. A background goroutine watches `commitIndex` and sends committed entries to `applyCh`.

---

## Step 1 — Leader advances `commitIndex` (20 min)

After a successful `AppendEntries` reply, the leader should check if any new index has been replicated on a majority. Add this as a new function:

```go
func (rf *Raft) maybeAdvanceCommitIndex() {
    // Try each index above current commitIndex
    for n := rf.commitIndex + 1; n < len(rf.log); n++ {
        // Only commit entries from current term (Figure 2, leader rule)
        if rf.log[n].Term != rf.currentTerm {
            continue
        }

        // Count how many peers have replicated up to n (including self)
        count := 1
        for i := range rf.peers {
            if i != rf.me && rf.matchIndex[i] >= n {
                count++
            }
        }

        // Majority reached → advance commitIndex
        if count >= len(rf.peers)/2+1 {
            rf.commitIndex = n
        }
    }
}
```

Then call it inside `sendHeartbeats`, in the `reply.Success` block, right after updating `matchIndex`:

```go
if reply.Success {
    newMatch := prevLogIndex + len(entries)
    if newMatch > rf.matchIndex[peer] {
        rf.matchIndex[peer] = newMatch
    }
    rf.nextIndex[peer] = rf.matchIndex[peer] + 1
    rf.maybeAdvanceCommitIndex()  // ← add this line
} else {
    ...
}
```

## Step 2 — Apply loop (20 min)

This goroutine watches `commitIndex` and sends newly committed entries to `applyCh`. Add it to `Make()` and pass `applyCh` in.

First, add `applyCh` to your struct:

```go
type Raft struct {
    // ... existing fields ...
    applyCh chan ApplyMsg
}
```

Initialize it in `Make()`:

```go
rf.applyCh = applyCh

// start apply loop
go rf.applyLoop()
```

Then the loop itself:

```go
func (rf *Raft) applyLoop() {
    for !rf.killed() {
        time.Sleep(10 * time.Millisecond)

        rf.mu.Lock()
        // Apply all committed but not yet applied entries
        for rf.lastApplied < rf.commitIndex {
            rf.lastApplied++
            msg := ApplyMsg{
                CommandValid: true,
                Command:      rf.log[rf.lastApplied].Command,
                CommandIndex: rf.lastApplied,
            }
            rf.mu.Unlock()
            rf.applyCh <- msg   // send outside lock — channel send can block
            rf.mu.Lock()
        }
        rf.mu.Unlock()
    }
}
```

The unlock/relock around the channel send is important — never hold the mutex while sending on a channel, it can deadlock.

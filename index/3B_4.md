## Accelerated Backtracking

The idea: instead of backing up one slot at a time, the follower tells the leader **which term conflicted and where that term starts**, so the leader can skip the entire bad term in one jump.

**Step 1 — Add conflict hint fields to `AppendEntriesReply`:**

```go
type AppendEntriesReply struct {
    Term          int
    Success       bool
    ConflictTerm  int  // term of the conflicting entry (-1 if log too short)
    ConflictIndex int  // first index of that conflicting term
}
```

**Step 2 — Fill them in `AppendEntries` on rejection:**

```go
// no entry at PrevLogIndex → log too short
if args.PrevLogIndex >= len(rf.log) {
    reply.ConflictTerm = -1
    reply.ConflictIndex = len(rf.log)
    return
}

// wrong term at PrevLogIndex → find where that term starts
if rf.log[args.PrevLogIndex].Term != args.PrevLogTerm {
    reply.ConflictTerm = rf.log[args.PrevLogIndex].Term
    // walk back to find first index of this conflicting term
    reply.ConflictIndex = args.PrevLogIndex
    for reply.ConflictIndex > 1 && rf.log[reply.ConflictIndex-1].Term == reply.ConflictTerm {
        reply.ConflictIndex--
    }
    return
}
```

**Step 3 — Use the hint in `sendHeartbeats` on failure:**

Replace your current `else` block:

```go
} else {
    if reply.ConflictTerm == -1 {
        // Follower log too short — jump directly to their log end
        rf.nextIndex[peer] = reply.ConflictIndex
    } else {
        // Search our log for the conflict term
        newIndex := -1
        for j := len(rf.log) - 1; j >= 1; j-- {
            if rf.log[j].Term == reply.ConflictTerm {
                newIndex = j + 1
                break
            }
        }
        if newIndex == -1 {
            // We don't have that term at all — jump to follower's first conflict index
            rf.nextIndex[peer] = reply.ConflictIndex
        } else {
            rf.nextIndex[peer] = newIndex
        }
    }
    if rf.nextIndex[peer] < 1 {
        rf.nextIndex[peer] = 1
    }
}
```

## Session 13: Fix All Index Arithmetic

This is the most tedious session but also the most important. Every place you use `len(rf.log)`, `rf.log[i]`, or raw indices needs to be updated. Go through each function systematically.

---

## The Rule

Replace every occurrence of:

| Old | New |
|-----|-----|
| `len(rf.log) - 1` | `rf.lastLogIndex()` |
| `rf.log[i]` | `rf.log[rf.logIndex(i)]` |
| `rf.log[len(rf.log)-1].Term` | `rf.logTerm(rf.lastLogIndex())` |
| `rf.log[ni:]` | `rf.log[rf.logIndex(ni):]` |
| `rf.log[:idx]` | `rf.log[:rf.logIndex(idx)]` |

---

## Fix each function

**`startElection` — building RequestVoteArgs:**
```go
args := &RequestVoteArgs{
    Term:         term,
    CandidateId:  rf.me,
    LastLogIndex: rf.lastLogIndex(),              // ← fix
    LastLogTerm:  rf.logTerm(rf.lastLogIndex()),  // ← fix
}
```

**`RequestVote` — log freshness check:**
```go
lastIndex := rf.lastLogIndex()          // ← fix
lastTerm := rf.logTerm(lastIndex)       // ← fix
logOk := args.LastLogTerm > lastTerm || (args.LastLogTerm == lastTerm && args.LastLogIndex >= lastIndex)
```

**`sendHeartbeats` — building AppendEntriesArgs:**
```go
ni := rf.nextIndex[peer]
prevLogIndex := ni - 1
prevLogTerm := rf.logTerm(prevLogIndex)              // ← fix
entries := make([]LogEntry, len(rf.log[rf.logIndex(ni):]))  // ← fix
copy(entries, rf.log[rf.logIndex(ni):])                     // ← fix
```

**`AppendEntries` — the append loop:**
```go
for i, entry := range args.Entries {
    idx := args.PrevLogIndex + 1 + i

    if idx < rf.lastLogIndex() + 1 {    // ← fix: was idx < len(rf.log)
        if rf.logTerm(idx) != entry.Term {           // ← fix
            rf.log = rf.log[:rf.logIndex(idx)]       // ← fix
            rf.log = append(rf.log, args.Entries[i:]...)
            break
        }
    } else {
        rf.log = append(rf.log, args.Entries[i:]...)
        break
    }
}
```

**`AppendEntries` — PrevLogIndex check:**
```go
// no entry at PrevLogIndex → reject
if args.PrevLogIndex < rf.lastIncludedIndex {
    // leader is behind our snapshot — this shouldn't happen normally
    reply.ConflictIndex = rf.lastIncludedIndex + 1
    reply.ConflictTerm = -1
    return
}

if args.PrevLogIndex > rf.lastLogIndex() {    // ← fix: was >= len(rf.log)
    reply.ConflictTerm = -1
    reply.ConflictIndex = rf.lastLogIndex() + 1
    return
}

if rf.logTerm(args.PrevLogIndex) != args.PrevLogTerm {  // ← fix
    reply.ConflictTerm = rf.logTerm(args.PrevLogIndex)  // ← fix
    reply.ConflictIndex = args.PrevLogIndex
    for reply.ConflictIndex > rf.lastIncludedIndex + 1 &&   // ← fix
        rf.logTerm(reply.ConflictIndex-1) == reply.ConflictTerm {
        reply.ConflictIndex--
    }
    return
}
```

**`maybeAdvanceCommitIndex`** — no change needed, it uses `commitIndex` and `matchIndex` which are absolute indices already.

**`applyLoop`:**
```go
for rf.lastApplied < rf.commitIndex {
    rf.lastApplied++
    msg := ApplyMsg{
        CommandValid: true,
        Command:      rf.log[rf.logIndex(rf.lastApplied)].Command,  // ← fix
        CommandIndex: rf.lastApplied,
    }
    ...
}
```

**`becomeLeader`:**
```go
for i := range rf.peers {
    rf.nextIndex[i] = rf.lastLogIndex() + 1  // ← fix
    rf.matchIndex[i] = 0
}
```
